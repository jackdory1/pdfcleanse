The hidden risk

Even if you cleaned it, can you prove it?

Every PDF you share carries hidden metadata. Most firms know this and run files through Adobe Acrobat Sanitize before sending — and Sanitize is a thorough first pass. What it doesn't produce is a record. No per-file certificate. No SHA-256 hash. No evidence the process ran. When a complaint lands under DUAA s.164A, the regulator asks one question: what process did you have in place? Verbal assurance is not the answer they're looking for.

Stage 1
Info dictionary
Author name, job title, creation date, software version, revision count. Written automatically by Word, Acrobat and Google Docs.
✗ Reveals author identity and internal tools
Stage 2
XMP packets
Adobe's extended metadata format embedded as XML inside the file body. Invisible in viewers. Fully readable by forensic software.
✗ Survives most standard cleaning tools
Stage 3
Document IDs
Two unique hexadecimal fingerprints in the PDF trailer. Link document versions. Can identify a file even after metadata removal.
✗ Missed by Smallpdf and most online tools
Stage 4
PieceInfo streams
Application workflow data written by InDesign, Illustrator and Acrobat. Contains internal project names, operator IDs and stage information.
✗ Rarely stripped by any competing tool

Questions

Frequently asked

Does the audit certificate hold up if something is missed?+
The honest answer: yes, for the purpose it's designed for. No metadata-removal tool can guarantee absolute cleanliness — not Adobe Sanitize, not PDF Cleanse, not any combination. What UK GDPR, the SRA Code, and the ICO actually require is that controllers can demonstrate appropriate technical and organisational measures. Controllers are assessed on whether they had a defensible process in place, not on whether that process was perfect in every edge case. The audit certificate is evidence of process, not a guarantee of outcome. It documents what was scanned, what was found, what was processed, the SHA-256 hash of input and output, the timestamp, and the tool version. It is explicit that manual verification in Adobe Acrobat is the recommended final step for high-stakes disclosure. The recommended compliance workflow: (1) run the file through Adobe Acrobat Sanitize — the primary cleaning pass; (2) run it through PDF Cleanse — independent verification, with a per-file audit certificate generated automatically; (3) for high-stakes disclosure, verify in Acrobat → File → Properties as a final human check; (4) retain the certificate in your compliance file alongside the disclosed document. If a regulator or court ever asks "what process did you have in place?", the certificate is your evidenced answer. Without one, the answer is verbal assurance — which is materially less defensible. This two-tool, two-pass approach with documented independent verification mirrors established compliance principles: independent attestation produces evidence that single-tool processes cannot.
Why use PDF Cleanse alongside Adobe Acrobat?+
Adobe Acrobat Pro is the standard tool for PDF preparation, and its Sanitize Document function handles the bulk of pre-disclosure cleaning — comments, attachments, scripts, hidden layers, XMP packets, PieceInfo streams, basic metadata. For most documents, Sanitize is a thorough first pass. PDF Cleanse is built for the second pass — independent verification with a per-file audit certificate. The two are complementary, not competing. Acrobat does the cleaning work. PDF Cleanse produces the documented evidence that Acrobat doesn't. If your firm already runs Acrobat, PDF Cleanse adds the audit-trail layer at £457 one-time (Office tier, 10 users). See the FAQ above on whether the certificate holds up if something is missed. The forensic defensibility brief sets out the architecture and certificate format in detail.

What happens to my files after I clean them?+
Nothing. The cleaned file is downloaded to your device. The original and cleaned versions exist only in browser memory during processing and are discarded when you close the tab. PDF Cleanse has no server, no database, and no storage. It is physically impossible for us to retain your files.
Will cleaning a PDF invalidate its digital signature?+
Yes, in most cases. Digital signatures in PDFs cryptographically cover the file's byte content — including metadata. Stripping metadata alters that content, which will invalidate any existing signature. If your document carries a digital signature that must remain verifiable, clean first, then re-sign the output.
Does PDF Cleanse remove GPS or location data?+
Not from embedded images. PDF Cleanse removes PDF-level structural metadata — Info dictionary, XMP packets, document IDs, and PieceInfo streams. If your PDF contains embedded images, those images may carry their own EXIF metadata including GPS coordinates. That data lives inside the image itself. If location data in embedded images is a concern, strip EXIF data from the images before embedding them.
